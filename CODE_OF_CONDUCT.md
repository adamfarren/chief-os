# Code of Conduct

## Our Commitment

Chief is an open-source project building AI skills and agents for executive operations. Contributors come from many backgrounds and disciplines. We are committed to making participation in this project a welcoming, respectful, and productive experience for everyone, regardless of background, identity, or experience level.

## Standards

### Expected Behavior

- **Be direct and constructive.** Give clear, actionable feedback. Disagree with ideas, not people.
- **Be rigorous.** Back claims with evidence. Flag uncertainty honestly.
- **Be inclusive.** Welcome newcomers. Explain context. Remember that contributors come from different disciplines.
- **Be responsible.** Consider the downstream impact of contributions on the people and organizations that rely on this tooling.
- **Be collaborative.** Share credit. Build on others' work generously. Assume good intent.

### Unacceptable Behavior

- Harassment, intimidation, or discrimination of any kind
- Personal attacks, insults, or derogatory language
- Publishing others' private information without explicit permission
- Trolling, derailing, or sustained disruption of discussions
- Any conduct that would be inappropriate in a professional setting

## Scope

This Code of Conduct applies to all project spaces — GitHub issues, pull requests, discussions, and any other forums associated with Chief. It also applies when representing the project in public spaces.

## Data-Handling Expectations

Chief skills read from and write to third-party services using the contributor's own credentials. Contributions should respect that responsibility:

1. **Never include real customer data, credentials, or PII** in issues, pull requests, examples, tests, or documentation. Use synthetic data only.
2. **Flag any change that could expand the blast radius** of a skill — new write paths, reduced confirmation gates, or new MCP scopes — so reviewers can weigh the tradeoff.
3. **Ask when unsure.** If a change might have security, privacy, or compliance implications in your environment, raise it in the PR description rather than assuming.

## Enforcement

Instances of unacceptable behavior may be reported by contacting the project maintainers. All reports will be reviewed and investigated promptly and fairly. Maintainers will maintain confidentiality with regard to the reporter.

### Enforcement Actions

Maintainers may take any action they deem appropriate, including:

- A private conversation to clarify expectations
- A formal warning
- Temporary or permanent ban from project spaces
